Three approaches to the upgrade

All three include the model swap, skeleton loader, e-ink flash, and HDRI + contact shadow. They differ in scope and material richness.

A

Targeted polish — Recommended

What: Swap to casefinal.glb, fix mesh detection (node names changed slightly), apply HDRI environment + contact shadow, silhouette skeleton on load, e-ink flash transition.

Materials: Keep the two-material system (dark matte case + e-ink screen). Extend it to handle the new mesh groups (board, pins, carabiner) with a single fallback dark material.

Trade-off: Fastest, lowest risk. The rich internal geometry (NeoPixels, PCB, carabiner) is visible but not specially highlighted — it all reads as dark hardware, which is honest to the device aesthetic.

B

Rich material system

What: Everything in A, plus per-group material treatment for the new mesh types: metallic pins/screws, PCB green for board nodes, emissive NeoPixel LEDs with a subtle glow, metallic carabiner.

Materials: 5–6 distinct materials keyed by node name prefix. The NeoPixels pulse gently in useFrame — a living detail visible on close inspection.

Trade-off: More visual depth and authenticity, but adds complexity to GlimpseModel.jsx and requires tuning each material against the HDRI. ~2× the implementation work.

C

Custom e-ink shader

What: Everything in A, but instead of a simple color flash for the e-ink transition, writes a custom ShaderMaterial that does a real scan-line wipe + white flash in GLSL — looks exactly like a hardware e-ink refresh.

Materials: Same two-material system as A, but the screen material becomes a custom shader with uniforms for progress, old texture, and new texture.

Trade-off: The most authentic screen transition, but GLSL shader adds fragility and complexity for a subtlety most users won't consciously notice. Best if the transition is a core part of the brand story.
