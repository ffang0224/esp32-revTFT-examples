Design: Glimpse Web 3D Experience Polish

Four targeted changes — each self-contained. Scope: GlimpseModel.jsx, App.jsx (canvas + Suspense), and a new ModelLoader component.

1 — Model swap + mesh detection fix

Change the import in GlimpseModel.jsx from case.glb → casefinal.glb. Update isCaseEInkScreenMesh to check the node's own name in addition to its parent's name — in the new model, e_ink_screen.* nodes are leaf meshes, not parent groups. All other new mesh types (board, pin, neopixel_*, carabiner, etc.) fall through to the dark matte fallback material.

2 — Silhouette skeleton loader

Replace the bare Suspense fallback={null} in App.jsx with a <ModelSkeleton> component rendered inside the Canvas. It's a simple dark box geometry (matching the device's approximate proportions) with a semi-transparent dark material — no texture, just shape. When the real model finishes loading, the skeleton fades out (opacity 0) over ~400ms via useFrame, then unmounts. The real model fades in simultaneously from opacity 0 → 1 over the same duration.

3 — HDRI environment + contact shadow

Add <Environment preset="studio" /> from @react-three/drei inside the Canvas. This replaces purely directional lighting with a physically-based environment map — the dark matte case picks up subtle ambient reflections that read as premium without being shiny. Keep the existing directional lights but reduce their intensities by ~30% to avoid over-brightening. Add <ContactShadows /> (opacity 0.5, blur 2.5, far 3) directly below the model's rest position — the floating bob animation will move the model away from the shadow, giving a natural depth cue.

4 — E-ink flash transition

In GlimpseModel.jsx's useFrame, when a texture change is detected (currentTexRef !== targetUrl), instead of swapping immediately: (1) flash the screen material color to #ffffff over ~80ms, (2) swap the texture at peak white, (3) fade back to normal over ~120ms. Implemented as a small state machine with a flashRef tracking progress and phase (idle / flash-in / swap / flash-out). No external dependencies needed — runs entirely in useFrame.
