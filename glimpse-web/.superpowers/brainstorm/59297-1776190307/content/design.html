Material system design

All changes are in GlimpseModel.jsx only. One consolidated useEffect replaces the existing two.

Architecture change

The existing two useEffects (screen mat + case mat) are replaced by one consolidated useEffect([scene]) that:

  1. Creates all 7 materials upfront
  2. Calls getMaterialForMesh(node) during scene.traverse to assign the right one
  3. Stores all materials in allMaterialsRef (array) for fade-in + disposal
  4. Keeps screenMaterialRef for the existing texture/flash logic

The fade-in block in useFrame iterates allMaterialsRef.current to set opacity on all transparent materials simultaneously. Non-transparent materials (LEDs, PCB, metal) skip the fade.

Transparent case rendering

The frosted case uses depthWrite: false — this prevents it from blocking the depth buffer so interior meshes always render through it, regardless of Three.js sort order. side: THREE.DoubleSide ensures inside faces of the shell are also visible.
